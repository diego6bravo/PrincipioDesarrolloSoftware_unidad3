# 🛠️ Proyecto Unidad 3: Control de Versiones con Git y GitHub

Este proyecto tiene como objetivo demostrar el dominio de herramientas de control de versiones y la gestión de proyectos colaborativos. Se enfoca en el uso de Git y la plataforma GitHub para el seguimiento de cambios, creación de ramificaciones y mantenimiento de repositorios.

## 📋 Descripción de la Actividad
En esta unidad se trabajó la configuración de un entorno de desarrollo profesional utilizando:
* **Configuración inicial:** Identidad del usuario y creación de repositorios.
* **Flujo de trabajo:** Gestión de estados (Modified, Staged, Committed).
* **Ramificaciones (Branching):** Creación de ramas para desarrollo de características independientes.
* **Sincronización:** Conexión entre el repositorio local y el remoto en GitHub.

---

## 🚀 Guía de Uso

### 1. Clonar el proyecto
Para obtener una copia exacta del proyecto en tu máquina local, escribe el siguiente comando en tu terminal:
```bash 
git clone https://github.com/diego6bravo/PrincipioDesarrolloSoftware_unidad3/edit/main/README.md
```

### 2. Contribuir
Si quieres contribuir a este proyecto te invitamos a que realices un *fork* del mismo y trabajes cambios desde allí. Posteriormente puedes enviar un *pull request* para que revisemos tus cambios y los podamos incluir en este proyecto.
